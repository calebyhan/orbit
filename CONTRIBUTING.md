## How to Contribute

### Documentation Improvements

If you'd like to improve documentation:

1. Fork the repository
2. Make your changes following the documentation conventions in `docs/README.md`
3. Ensure all `.md` files use the italic date format: `*Last edited: YYYY-MM-DD*`
4. Add appropriate "Related Files" sections to new documentation
5. Submit a pull request

### Code Contributions

For code contributions:

1. Review the project scope in `docs/01-overview/project_scope.md`
2. Check existing test plans in `docs/98-test-plans/`
3. Follow the module specifications in `docs/`
4. Write tests for new functionality
5. Submit a pull request with clear description

### Reporting Issues

Found a bug or have a suggestion? Please open an issue on GitHub with:

* Clear description of the problem or enhancement
* Steps to reproduce (for bugs)
* Expected vs. actual behavior
* Relevant configuration and environment details

---

## Documentation Standards

All documentation contributions should follow these conventions:

* Use italic markdown date format: `*Last edited: YYYY-MM-DD*`
* Include "Related Files" sections with cross-references
* Provide clear structure with purpose, inputs, outputs, procedures
* Include acceptance criteria and executable examples where applicable

See `docs/README.md` for complete documentation guidelines.

---

## Code of Conduct

* Be respectful and constructive
* Focus on the technical merits of contributions
* Help maintain clear, accurate documentation
* Follow existing code style and conventions
